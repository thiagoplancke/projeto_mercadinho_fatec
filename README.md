# 🛒 Mercadinho Autônomo – FATEC Rio Claro

Projeto acadêmico para criação de um **sistema de mercadinho autônomo** dentro da FATEC Rio Claro.

O objetivo do projeto é digitalizar o funcionamento do mercadinho interno utilizado pelos alunos, que atualmente opera de forma manual: os estudantes pegam produtos, anotam em uma folha e realizam o pagamento via Pix.

O sistema busca tornar esse processo **mais organizado, rastreável e automatizado**, além de servir como base para experimentos com **visão computacional e inteligência artificial**.

---

# 📌 Problema

Atualmente o mercadinho funciona com um sistema baseado em confiança:

* os alunos pegam produtos da prateleira
* anotam manualmente o que compraram
* realizam o pagamento via Pix

Esse modelo possui alguns problemas:

* erros ou esquecimento ao registrar compras
* dificuldade para acompanhar estoque
* falta de histórico estruturado de vendas
* ausência de automação no processo

---

# 💡 Proposta

Desenvolver um **sistema de mercadinho autônomo** que permita:

* cadastro de alunos
* cadastro de produtos
* seleção de produtos comprados
* cálculo automático do valor da compra
* geração de pagamento via Pix
* registro de histórico de compras

O projeto também servirá como **base experimental para aplicações de Inteligência Artificial**, incluindo reconhecimento facial e automação de processos.

---

# 🧠 Tecnologias utilizadas

Inicialmente o projeto será desenvolvido inteiramente em **Python**, priorizando simplicidade e rapidez de prototipagem.

Principais ferramentas planejadas:

* **Python**
* **Tkinter** – interface gráfica
* **SQLite** – banco de dados local
* **OpenCV** – visão computacional (futuras versões)
* **Bibliotecas de QR Code / Pix** – geração de pagamento

---

# 🏗️ Estrutura inicial do sistema

O sistema será dividido em alguns componentes principais:

* **Cadastro de alunos**
* **Cadastro de produtos**
* **Seleção de compras**
* **Registro de histórico**
* **Banco de dados**

Versões futuras irão integrar **visão computacional e reconhecimento facial**.

---

# 🚀 Roadmap do projeto

O projeto será desenvolvido em etapas progressivas.

## Nível 1 – Sistema básico

Primeira versão funcional do mercadinho.

Funcionalidades:

* cadastro de alunos
* cadastro de produtos
* seleção manual de produtos
* cálculo automático da compra
* armazenamento em banco de dados

Objetivo: criar um **sistema funcional simples para registrar compras**.

---

## Nível 2 – Identificação do aluno

Introdução de um sistema de identificação mais automatizado.

Possíveis melhorias:

* identificação por ID
* login rápido
* preparação do banco de dados para reconhecimento facial

---

## Nível 3 – Reconhecimento facial

Integração com **visão computacional** para identificar automaticamente o aluno.

Fluxo esperado:

1. aluno se aproxima do sistema
2. câmera reconhece o rosto
3. perfil do aluno é carregado automaticamente
4. aluno seleciona produtos

Tecnologias possíveis:

* OpenCV
* bibliotecas de face recognition

---

## Nível 4 – Automação do processo de compra

Evolução do sistema para tornar o processo cada vez mais automático.

Possibilidades futuras:

* geração automática de QR Code Pix
* histórico detalhado de compras
* painel de análise de vendas
* alertas de estoque baixo

---

# 📊 Possíveis evoluções

O projeto também pode evoluir para incluir:

* análise de produtos mais vendidos
* previsão de estoque
* dashboard de vendas
* integração com aplicativos mobile

---

# 🎓 Objetivo acadêmico

Além de resolver um problema real dentro da faculdade, o projeto também tem como objetivo explorar conceitos de:

* engenharia de software
* estruturação de sistemas
* banco de dados

* inteligência artificial aplicada
